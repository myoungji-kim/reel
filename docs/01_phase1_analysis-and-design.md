# Phase 1 — 분석 및 설계

## 사전 준비

`docs/00_context_project-overview.md` 와 `docs/prototype_reel.html` 을 먼저 읽어줘.

---

## 역할

너는 시니어 풀스택 개발자야.
`docs/prototype_reel.html` 의 **기능과 디자인을 100% 유지**하면서, 프로덕션에서 유지보수 가능한 구조로 프로젝트화할 거야.

이번 Phase에서는 **코드를 작성하지 말고**, 아래 항목을 텍스트로만 보여줘.

---

## 파악해야 할 기능 목록

프로토타입 코드를 분석해서 아래 기능들이 모두 존재하는지 확인해줘:

- 상단 탭 전환 (오늘의 현장 ↔ 필름 롤)
- 필름 퍼포레이션 스크롤 애니메이션 (40s linear infinite)
- 채팅: 메시지 입력/전송, AI 응답, 타이핑 인디케이터
- 메시지 3개 이상 시 현상 배너 표시
- 현상하기: DEVELOPING 오버레이(진행바 포함) → AI 일기 생성 → 미리보기 오버레이
- 미리보기에서 제목/본문 직접 수정 가능
- 최종 저장 후 필름 롤에 프레임 추가 + 탭 자동 전환
- 프레임 클릭 시 상세 오버레이
- Anthropic API 연동 (채팅 응답 + 일기 현상)
- 에러 시 fallback 처리

## 설계 산출물 (텍스트로만 작성)

### 1. 관심사 분리 목록
현재 단일 HTML 파일에서 분리되어야 할 관심사를 나열해줘.

### 2. 디렉토리 구조
프론트엔드 / 백엔드 전체 디렉토리 구조를 트리 형태로 보여줘.

### 3. 개선 필요 사항
기존 코드에서 발견된 문제점 (에러 처리, 타입 안전성, 중복 코드 등).

### 4. DB 스키마
테이블 DDL을 작성해줘.

### 5. API 명세
Method / URL / 설명 표로 정리해줘.
