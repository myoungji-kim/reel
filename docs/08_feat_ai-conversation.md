# AI 대화 품질 고도화 (AI Conversation Quality)

## 사전 준비
- `docs/00_context_project-overview.md` 읽기
- 관련 파일:
  - `backend/src/main/java/com/reel/ai/AiService.java` (또는 `ChatService.java`)
  - `backend/src/main/java/com/reel/chat/ChatService.java`
  - AI 시스템 프롬프트가 정의된 상수/설정 파일

## 기능 개요
AI가 단순 응답에 그치지 않고, 사용자의 감정·구체적 장면·사람·장소를 자연스럽게 끌어내는 후속 질문을 포함하도록 시스템 프롬프트를 개선한다.

## UX 의도
현재 AI는 사용자 말에 공감은 하지만 더 깊이 파고드는 질문이 부족하다.
"오늘 기분이 어땠어요?"처럼 형식적인 게 아니라, 대화 맥락을 읽고 "그 사람이 뭐라고 했을 때 가장 마음이 찡했어요?"처럼 구체적이고 감성적인 질문을 던져야 한다.
이렇게 모인 디테일이 나중에 현상된 일기를 더 생생하게 만든다.

## 구현 요청

### 백엔드

1. **AI 시스템 프롬프트 수정**
   - 채팅용 시스템 프롬프트에 아래 지침 추가:

   ```
   [후속 질문 지침]
   - 사용자 메시지에서 감정, 구체적 장면, 등장인물(사람·동물), 장소·날씨·소리 등 감각적 요소가 언급되면 반드시 그 부분을 더 구체화하는 질문을 포함할 것
   - 질문은 1개로 제한. 여러 질문을 연달아 쏟아내지 말 것
   - 이미 해당 정보가 충분히 나온 경우(대화 3회 이상 축적된 맥락) 질문 생략 가능
   - 질문 어조: 캐주얼하고 따뜻하게. 인터뷰나 심문처럼 느껴지지 않을 것
   - 예시 (참고용, 그대로 쓰지 말 것):
     - "그때 어떤 감정이 들었어요? 조금 더 얘기해줄 수 있어요?"
     - "그 장소가 어디였어요? 분위기가 궁금해요."
     - "그 사람이 뭐라고 했을 때 가장 마음에 남았어요?"
   ```

2. **대화 맥락 판단 로직 추가** (선택적 개선)
   - 현재 대화 메시지 수(또는 총 글자 수)가 임계값(예: 메시지 5개 이상, 또는 사용자 총 입력 300자 이상) 초과 시, 시스템 프롬프트에 "충분한 맥락이 확보됨. 추가 질문보다 공감과 마무리에 집중할 것" 힌트를 추가
   - 이 로직은 `ChatService`에서 메시지 리스트 길이로 판단

3. **현상(일기 생성)용 프롬프트는 별도 유지**
   - 채팅용 프롬프트와 현상용 프롬프트가 혼용되지 않도록 분리 확인
